# LGTM HQ

**Building developer tools that look good, work well, and ship fast.**

We are an open-source organization focused on developer experience. Our projects span
linting, theming, CI/CD automation, and developer environment tooling -- all designed to
reduce friction and raise the bar for code quality.

## Projects

| Project | Description |
| --- | --- |
| [py-lintro](https://github.com/lgtm-hq/py-lintro) | Unified CLI for code formatting, linting, and quality assurance |
| [turbo-themes](https://github.com/lgtm-hq/turbo-themes) | Universal, accessible theme packs and a drop-in theme selector |
| [lgtm-ci](https://github.com/lgtm-hq/lgtm-ci) | Reusable CI/CD components: composite actions, workflows, and shell libraries |
| [Rustume](https://github.com/lgtm-hq/Rustume) | Privacy-first, offline-first resume builder powered by Rust and Typst |
| [holy-grail](https://github.com/lgtm-hq/holy-grail) | Comprehensive developer installation and setup guides |
| [homebrew-tap](https://github.com/lgtm-hq/homebrew-tap) | Homebrew tap for installing LGTM tools |

## Contributing

We welcome contributions across all repositories. Each project has its own setup
instructions, but the fundamentals are the same everywhere:

- **Conventional Commits** for all commit messages
- **Lintro** for linting and formatting (where applicable)
- **Squash merges** so PR titles drive the changelog

See any repository's `CONTRIBUTING.md` for project-specific details, or read the
[org-wide contributing guide](https://github.com/lgtm-hq/.github/blob/main/CONTRIBUTING.md).

# Support

## How to Get Help

Looking for help with an LGTM HQ project? Here are the best ways to get support.

### Documentation

Start with the project's README and docs:

| Project | Documentation |
| --- | --- |
| [py-lintro](https://github.com/lgtm-hq/py-lintro) | README and `docs/` directory |
| [turbo-themes](https://github.com/lgtm-hq/turbo-themes) | README and example projects |
| [lgtm-ci](https://github.com/lgtm-hq/lgtm-ci) | README and action/workflow docs |
| [Rustume](https://github.com/lgtm-hq/Rustume) | README and crate-level docs |
| [holy-grail](https://github.com/lgtm-hq/holy-grail) | Setup guides on the site |
| [homebrew-tap](https://github.com/lgtm-hq/homebrew-tap) | README |

### GitHub Issues

If the documentation does not answer your question:

1. **Search existing issues** in the relevant repository -- someone may have already
   asked the same question.
2. **Open a new issue** using the repository's issue templates. Choose the appropriate
   template (bug report, feature request, or question).

### Security Issues

For security vulnerabilities, **do not** open a public issue. See
[SECURITY.md](SECURITY.md) for responsible disclosure instructions.

## What to Include in a Support Request

To help us help you quickly, please provide:

- **Project and version** you are using.
- **Operating system** and relevant runtime versions (Python, Node.js, Rust, etc.).
- **Steps to reproduce** the issue, with commands and output.
- **Expected vs. actual behavior**.
- Any relevant **configuration files** or **error logs**.

## Response Times

LGTM HQ projects are maintained by volunteers. We do our best to respond promptly, but
please allow a few business days for a reply. Critical security issues are prioritized --
see [SECURITY.md](SECURITY.md) for expected timelines.

## What This Support Channel Is Not

- **Not a guaranteed SLA.** These are open-source projects maintained in good faith.
- **Not for third-party tool issues.** If the problem is in an upstream dependency
  (e.g., Ruff, Prettier, Typst), please report it to that project instead.
